

Some SAM FLAGS describe paired-end properties

Some SAM optional fields describe more paired-end properties: The last several fields of each SAM record usually contain SAM optional fields, which are simply tab-separated strings conveying additional information about the reads and alignments. A SAM optional field is formatted like this: "XP:i:1" where "XP" is the TAG, "i" is the TYPE ("integer" in this case), and "1" is the VALUE. See the SAM specification for details regarding SAM optional fields.

YF:i -> SAM optional field will be set to indicate the reason the read was filtered.
